You are updating the Astro documentation sidebar for nara.

**Job**: Add, remove, or reorganize entries in the Starlight sidebar configuration.

**Process**:

1. Read `docs/astro.config.mjs` to see the current sidebar structure

2. Check what documentation files exist:
   ```bash
   ls docs/src/content/docs/
   ls docs/src/content/docs/spec/
   ls docs/src/content/docs/concepts/
   ls docs/src/content/docs/api/
   ls docs/src/content/docs/running-nara/
   ```

3. Update the sidebar in `docs/astro.config.mjs`:
   - Add new entries for new documentation files
   - Remove entries for deleted files
   - Reorganize as needed

**Sidebar structure**:

The sidebar is organized into sections:
- **Overview** - Main entry point
- **Running Nara** - Deployment guides
- **Specification** - Technical specs (spec/*.md)
- **Concepts** - Conceptual guides (concepts/*.md)
- **API Reference** - Auto-generated from Go code (api/*.md)

**Entry format**:
```javascript
{ label: 'Human Readable Name', link: '/path/to/file/' }
```

Note: The link path is relative to `docs/src/content/docs/` and should end with `/`.

**Example - adding a new spec**:
```javascript
// In the Specification section's items array:
{ label: 'New Feature', link: '/spec/new-feature/' }
```

**Common tasks**:
- New spec file created → Add to Specification section
- New concept doc created → Add to Concepts section
- Renamed a file → Update the link path
- Reorganizing → Move entries between sections, reorder

**Auto-generated API docs**:
The API Reference section entries should match files in `docs/src/content/docs/api/`:
```javascript
{
  label: 'API Reference',
  items: [
    { label: 'Messages', link: '/api/messages/' },
    { label: 'Runtime', link: '/api/runtime/' },
    { label: 'Utilities', link: '/api/utilities/' }
  ]
}
```

These are generated by `scripts/gen-api-docs.sh` from Go packages: messages/, runtime/, utilities/.

**Verification**:
After updating, run:
```bash
make build-web
```

This builds the docs site and will error if links are broken.

**Deliverable**: Updated `docs/astro.config.mjs` with correct sidebar entries.
